🔄 LOADING INDICATOR & STATUS:

  IDLE State:
  └─ Status: IDLE (menunggu gesture)
  
  
  Saat Ada Gesture:
  └─ Status: SENDING... (sedang mengirim)
      └─ Background thread mengirim POST request
  
  
  Jika Berhasil:
  └─ Status: SUCCESS ✓ (ditampilkan 3 detik)
      └─ Warna: HIJAU
      └─ Auto-reset ke IDLE setelah 3 detik
  
  
  Jika Gagal:
  ├─ Status: ERROR ✗ (connection error)
  ├─ Status: TIMEOUT ✗ (request timeout)
  └─ Status: ERROR 500 (HTTP error dengan code)
      └─ Warna: MERAH
      └─ Auto-reset ke IDLE setelah 3 detik

═══════════════════════════════════════════════════════════════════════════════════

🚦 KONTROL TRAFFIC (PREVENT OVERLOAD):

  1. COOLDOWN CONTROL:
     ┌─ Gesture A (Kode:1) → Kirim ✓
     └─ 3 detik cooldown
        └─ Gesture A lagi → Ditolak (masih dalam cooldown)
        └─ Gesture B (Kode:2) → Kirim ✓ (kode berbeda)

  
  2. QUEUE SYSTEM:
     ┌─ Request 1 → Masuk queue + kirim
     ├─ Request 2 → Masuk queue + kirim (jika tidak timeout)
     ├─ Request 3 → Masuk queue + kirim
     ├─ Request 4 → Masuk queue + kirim
     ├─ Request 5 → Masuk queue + kirim
     └─ Request 6 → DITOLAK! (queue penuh)
         └─ Tunggu ada slot di queue sebelum bisa kirim
  
  
  3. TIMEOUT CONTROL:
     └─ Request → Kirim ke server
         └─ Tunggu 5 detik untuk response
             ├─ Jika dapat response → Sukses
             └─ Jika tidak → Timeout (dicatat di history)

═══════════════════════════════════════════════════════════════════════════════════

🔒 THREAD-SAFE MECHANISM:

  Main Thread (Video Processing):
  ├─ Baca frame dari kamera
  ├─ Deteksi hand gesture
  ├─ Update UI
  └─ Lock untuk akses monitor (brief)

  
  Background Thread (Network I/O):
  ├─ Ambil request dari queue
  ├─ POST ke server (non-blocking)
  ├─ Update status
  └─ Release lock setelah selesai
  
  
  Result:
  ✓ Kamera tidak freeze saat mengirim data
  ✓ Tidak ada race condition
  ✓ Data konsisten

📊 DATA STRUCTURE:

  DataMonitor Class:
  {
    send_queue: Queue(maxsize=5)         ← Thread-safe queue
    history: deque(maxlen=15)            ← Bounded history
    current_status: "SUCCESS ✓"          ← Real-time status
    last_sent_time: 1716637722.123       ← Timestamp
    last_sent_kode: 2                    ← Last gesture code
    lock: threading.Lock()               ← Thread safety
  }
  
  
  History Event Format:
  {
    'time': "14:35:22",                  ← HH:MM:SS
    'kode': 2,                           ← Gesture code (1-4)
    'fingers': 3,                        ← Finger count
    'status': "SENT"                     ← Event status
  }
